# 007. 데이터 저장 구조

## 핵심 설계: shared_products 공유 가격 구조
같은 쿠팡 상품을 여러 사용자가 추적할 때, 가격 데이터를 중복 저장하지 않고 공유한다.
- **shared_products**: 상품 가격/메타 정보 (봇이 관리, 전체 사용자 공유)
- **users/{uid}/items**: 사용자별 설정만 (목표가, 재구매 등)

## 저장소 이원화
| 저장소 | 역할 | 키/경로 |
|--------|------|---------|
| AsyncStorage (Zustand persist) | 로컬 캐시, 오프라인 사용 | `aigo-storage` |
| Firestore | 원격: 공유 가격 + 유저 설정 | 아래 참고 |

## Firestore 구조

### shared_products/{productId}
> 모든 사용자가 공유하는 상품 가격/메타 데이터.
> 가격 체크 봇이 주기적으로 업데이트한다.

```
productId: string               // 쿠팡 상품 ID (문서 ID와 동일)
vendorItemId?: string            // 판매자 옵션 ID
productName: string
category?: BabyCategory
thumbnail: string
url: string                      // www.coupang.com 직접 URL
currentPrice: number
priceHistory: { date, price }[]  // 최대 30일
trackerCount: number             // 추적 중인 사용자 수
lastCheckedAt: string            // ISO datetime (봇 마지막 체크)
createdAt: string                // ISO datetime
```

### users/{uid}
> 사용자 계정 정보 + 앱 설정

```
expoPushToken: string
notificationEnabled: boolean
repurchaseNotificationEnabled: boolean
isWowMember: boolean
babyBirthDate: string | null
lastActiveAt: string             // ISO datetime
```

### users/{uid}/items/{productId}
> 사용자별 추적 설정만 저장. 가격/메타 데이터는 shared_products 참조.

```
productId: string                // → shared_products/{productId} 참조키
affiliateUrl: string             // 이 사용자의 제휴 딥링크
targetPrice?: number
repurchaseEnabled?: boolean
repurchaseDays?: number
lastPurchasedAt?: string
createdAt: number
```

## 로컬 (Zustand + AsyncStorage)

### AppState
```
isWowMember: boolean
notificationEnabled: boolean
repurchaseNotificationEnabled: boolean
hasSeenOnboarding: boolean
babyBirthDate: string | null
trackedItems: TrackedItem[]      // 조합된 뷰 (shared + user 병합)
```

### TrackedItem (로컬 병합 뷰)
> 앱에서 사용하는 모델. Firestore에서 읽을 때 shared_products + user items를 조합.

```
id: string                       // productId
url: string                      // 사용자별 제휴 딥링크
resolvedUrl?: string             // www.coupang.com URL (shared에서)
productId?: string
vendorItemId?: string
productName: string              // shared에서
category?: BabyCategory          // shared에서
currentPrice: number             // shared에서
targetPrice?: number             // user에서
thumbnail: string                // shared에서
priceHistory: { date, price }[]  // shared에서
createdAt: number                // user에서
repurchaseEnabled?: boolean      // user에서
repurchaseDays?: number          // user에서
lastPurchasedAt?: string         // user에서
```

## 데이터 흐름

### 상품 등록
```
1. 사용자가 상품 URL 공유
2. 스크래핑 → productId 추출
3. shared_products/{productId} 존재 확인
   - 없으면: 신규 문서 생성 (가격/메타/trackerCount=1)
   - 있으면: trackerCount +1
4. users/{uid}/items/{productId} 생성 (목표가, 제휴링크 등)
5. 로컬 trackedItems에 병합 뷰 추가
```

### 가격 체크 봇 (3회/일)
```
1. shared_products에서 trackerCount > 0 인 상품 조회
2. 각 상품 가격 체크 → currentPrice, priceHistory 업데이트
3. 가격 변동 감지 시:
   - 해당 상품을 추적 중인 users 조회
   - 각 사용자의 targetPrice와 비교
   - 조건 충족 시 FCM 푸시 발송
```

### 상품 삭제
```
1. users/{uid}/items/{productId} 삭제
2. shared_products/{productId}.trackerCount -1
3. trackerCount === 0 이면 shared_products 문서도 삭제 (또는 유지 후 TTL)
```

### 앱 포그라운드 복귀
```
1. users/{uid}/items 조회 → productId 목록 확보
2. shared_products에서 해당 productId들 조회
3. 병합하여 로컬 trackedItems 업데이트
```

## 동기화 전략
1. **쓰기**: 로컬 먼저 → Firestore 비동기 (fire-and-forget)
2. **읽기**: shared_products + user items 병합 → 로컬 덮어쓰기
3. **가격 업데이트**: 봇이 shared_products만 업데이트 (사용자별 items 터치 안 함)
4. **유저 설정**: 변경 시 users/{uid} 및 users/{uid}/items/{productId}에 merge

## 장점
- **중복 제거**: 100명이 같은 기저귀를 추적해도 가격 체크 1회
- **봇 효율**: shared_products만 순회 (사용자 수 무관)
- **일관성**: 모든 사용자가 동일 가격 데이터 참조
- **비용 절감**: Firestore 읽기/쓰기 횟수 대폭 감소

## 보안 규칙 (firestore.rules 업데이트 필요)
```
shared_products/{productId}:
  - read: 인증된 사용자 누구나
  - create: 인증된 사용자 (새 상품 등록 시)
  - update: 인증된 사용자 (trackerCount 증감) + Admin SDK (봇 가격 업데이트)
  - delete: Admin SDK only

users/{uid}: 본인만 읽기/쓰기
users/{uid}/items/{itemId}: 본인만 읽기/쓰기
```
