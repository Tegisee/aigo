## [온보딩]

### 1. 생년월일 연/월/일 모두 입력

- **문제/기획 의도**: 월령 계산, 기념일(100일/돌/생일) 알림, D-Day 표시를 위해 정확한 생년월일이 필요. 연/월만으로는 일 단위 계산 불가.
- **현재 상태**: 온보딩에서 연/월만 입력받음. 일(day) 필드 없음.
- **구현 방향**: DatePicker 또는 연/월/일 3단 스크롤 피커로 교체. `useAppStore`의 `babyBirthDate` 필드를 `YYYY-MM-DD` 포맷으로 저장. 기존 `YYYY-MM` 데이터는 마이그레이션 시 `-01` 기본값 부여 후 설정에서 수정 유도.
- **우선순위**: 🔴 높음 (기념일/D-Day/월령 기능 전체의 전제조건)

### 2. 아이 이름/애칭/태명 입력 필드 추가

- **문제/기획 의도**: 홈화면 인사말, 공유 메시지, 알림 문구에 아이 이름을 사용하여 감성적 UX 제공. "쪼꼬미 25개월" 같은 개인화 표시.
- **현재 상태**: 이름 입력 필드 없음. 홈화면에 "아이고 3세"로 고정 표시.
- **구현 방향**: 온보딩 화면에 텍스트 입력 필드 추가 (placeholder: "예: 쪼꼬미, 콩이, 하늘이"). `useAppStore`에 `babyName: string` 필드 추가. 미입력 시 기본값 "우리 아이".
- **우선순위**: 🔴 높음 (7번 홈화면 표시, 23번 공유 메시지의 전제조건)

### 3. 아이 성별 입력 필드 추가

- **문제/기획 의도**: 성별 기반 추천 상품 필터링 (의류/신발 등), 향후 맞춤형 큐레이션에 활용.
- **현재 상태**: 성별 입력 없음.
- **구현 방향**: 온보딩에 선택 버튼 추가 (남아/여아/비공개). `useAppStore`에 `babyGender: 'male' | 'female' | 'unknown'` 필드 추가. 비공개 선택 시 성별 무관 추천.
- **우선순위**: 🟡 중간 (MVP에서는 필수 아님, Phase 2 카테고리/추천 시 활용)

### 4. 온보딩 예시 상품 육아용품으로 교체

- **문제/기획 의도**: 지금이야에서 fork한 에어팟프로 예시가 그대로 남아있음. 육아용품 앱 정체성에 맞지 않음.
- **현재 상태**: `OnboardingScreen.tsx`에 에어팟프로 예시 사용 중.
- **구현 방향**: 예시 상품을 육아용품으로 교체. 예: 하기스 기저귀 4단계 (공유 URL 시연), 압솔루트 분유 800g (가격 추적 시연). 썸네일 이미지도 육아용품으로 교체.
- **우선순위**: 🔴 높음 (앱 첫인상 직결, 간단한 텍스트/이미지 교체)

### 5. 준비완료 화면 예시 상품 육아용품으로 교체

- **문제/기획 의도**: 온보딩 마지막 "준비완료" 화면의 예시 상품도 에어팟프로 등 범용 상품이 남아있음.
- **현재 상태**: 지금이야 fork 상태 그대로.
- **구현 방향**: 4번과 동일하게 육아용품 예시로 교체. 카테고리 아이콘도 육아 감성에 맞게 수정.
- **우선순위**: 🔴 높음 (4번과 함께 일괄 작업)

### 6. 공유하기 시 앱 아이콘 교체

- **문제/기획 의도**: Share Extension에서 표시되는 앱 아이콘이 최종 확정 아이콘이어야 함.
- **현재 상태**: 임시 아이콘 사용 중. 최종 아이콘 미확정.
- **구현 방향**: 아이콘 확정 후 `assets/icon.png`, `assets/icon_android_512.png` 교체. Share Extension 아이콘도 함께 업데이트.
- **우선순위**: 🟢 낮음 (아이콘 확정 후 진행)

## [홈화면]

### 7. 상단 좌측 표시 형식 변경

- **문제/기획 의도**: "아이고 3세" → 아이 닉네임 + 월령 + 태어난 일수로 변경하여 개인화된 감성 UX 제공. 예: "쪼꼬미 25개월 · 태어난 지 762일째"
- **현재 상태**: 앱 이름 + 연령만 표시. 닉네임/일수 미표시.
- **구현 방향**: `babyName` + `babyBirthDate`로 월령(개월 수)과 일수(D+N) 계산. `dayjs` 또는 네이티브 Date 사용. 1번, 2번 완료 후 구현 가능.
- **우선순위**: 🔴 높음 (앱 핵심 아이덴티티. 1번, 2번 의존)

#### 기념일 D-Day 표시

- **문제/기획 의도**: 아이 기념일 D-7부터 홈 상단에 D-Day 배너 자동 노출. 당일엔 축하 메시지. 부모에게 감성적 가치 제공.
- **현재 상태**: 미구현.
- **구현 방향**:
  - D-Day 기준 기념일 목록:
    - 100일, 200일, 300일
    - 365일 (돌)
    - 돌 이후 500일 단위 (500, 1000, 1500...)
    - 특별 숫자: 1111일, 2222일, 3333일, 4444일
    - 매년 생일
  - `babyBirthDate` 기준으로 다가오는 기념일 계산
  - D-7부터 배너 노출, 당일 지나면 자동 숨김
  - 배너 문구 예시:
    - "쪼꼬미 100일까지 D-3!"
    - "오늘은 쪼꼬미 1000일이에요! 축하해요"
- **우선순위**: 🟡 중간 (7번 기본 표시 완료 후 추가)

### 8. 상단 우측 로그인/회원가입 버튼

- **문제/기획 의도**: 기본 익명 사용 유지하되, 데이터 보존을 원하는 사용자에게 소셜 로그인 연동 옵션 제공.
- **현재 상태**: 익명 Auth만 사용. 로그인 UI 없음.
- **구현 방향**: 홈 상단 우측에 프로필 아이콘 버튼 → 탭 시 로그인 바텀시트 (구글/카카오 선택). 미연동 상태에서는 경고 배지 + 안내 문구: "앱 삭제 또는 기기 변경 시 모든 데이터가 초기화됩니다." 25번과 연동.
- **우선순위**: 🟡 중간 (25번 로그인 구현과 함께 진행)

### 9. 오늘의 특가 → 육아 키워드 필터링 + 월령별 매칭

- **문제/기획 의도**: 골드박스 전체 상품 중 육아 관련 상품만 필터링하고, 아이 월령에 맞는 카테고리 우선 노출.
- **현재 상태**: 골드박스 전체 상품 무분별 표시 (범용 상품 포함).
- **구현 방향**: 골드박스 API 응답에서 육아 키워드(기저귀, 분유, 물티슈, 유아, 아기, 키즈 등) 필터링. 12번 월령별 카테고리 기준으로 매칭 점수 부여 후 정렬. 육아 상품이 부족할 경우 "전체 특가 더보기" 링크 제공.
- **우선순위**: 🟡 중간 (12번 카테고리 체계 완료 후)

### 10. "추적상품 가져오기" → "관심상품 가져오기" 문구 수정

- **문제/기획 의도**: 16번 단어 통일("추적" → "관심")의 일환. 부제목도 아이고 감성에 맞게 변경.
- **현재 상태**: "추적상품 가져오기" 문구 사용 중.
- **구현 방향**: 문구 교체 + 부제목 변경. 예: "쪼꼬미에게 필요한 상품을 찾아보세요" (`babyName` 변수 사용). 16~18번과 일괄 작업.
- **우선순위**: 🔴 높음 (단순 텍스트 변경, 16~18번과 함께)

### 11. 카테고리 필터 텍스트 박스 하단 잘림 버그

- **문제/기획 의도**: 카테고리 칩의 텍스트가 하단이 잘려서 표시됨. 가독성 저하.
- **현재 상태**: `index.tsx`의 `categoryChips` 스타일에서 칩 높이 부족.
- **구현 방향**: 칩 컴포넌트의 `paddingVertical` 확대 또는 `minHeight` 설정. 텍스트가 완전히 보이도록 높이 조정.
- **우선순위**: 🔴 높음 (UI 버그, 간단 수정)

### 12. 카테고리 월령 기반 동적 변경

- **문제/기획 의도**: 아이 월령에 따라 필요한 카테고리가 다름. 고정 목록 대신 월령별 맞춤 카테고리 제공이 아이고의 핵심 차별점.
- **현재 상태**: `types/index.ts`에 9개 고정 카테고리 (`BabyCategory`). 월령 분기 없음.
- **구현 방향**:
  - 월령 구간별 카테고리 매핑 정의:
    - 0~6개월: 기저귀, 분유, 수유용품, 물티슈, 속싸개/배냇저고리, 신생아 스킨케어, 기타
    - 7~12개월: 기저귀, 이유식/이유식도구, 수유용품, 물티슈, 장난감(모빌/치발기), 보행기/점퍼루, 기타
    - 13~24개월: 기저귀, 유아식, 장난감, 의류, 신발, 안전용품(모서리보호대 등), 기타
    - 25~48개월: 의류, 장난감, 유모차/카시트, 신발, 유아 도서/학습, 생활용품, 기타
    - 49개월~7세: 장난감, 의류, 신발, 가구(책상/의자), 학습교구, 도서, 스포츠용품, 기타
    - 8~13세: 의류, 신발, 학용품, 책가방, 도서, 스포츠용품, 전자기기, 기타
  - `getCategoriesByMonth(monthAge: number)` 함수 구현
  - `BabyCategory` 타입을 확장하여 전체 카테고리 포함
  - 1번(생년월일) 완료 후 월령 자동 계산 → 카테고리 동적 변경
- **우선순위**: 🔴 높음 (아이고 핵심 차별화. 1번 의존)

### 13. 카테고리 탭 클릭 시 추천 상품 리스트 표시

- **문제/기획 의도**: 카테고리 탭 클릭해도 아무 반응 없음. 해당 카테고리 + 월령에 맞는 추천 상품을 보여줘야 함.
- **현재 상태**: 카테고리 칩 클릭 시 등록된 관심상품만 필터링. 추천 상품 리스트 없음.
- **구현 방향**:
  - 쿠팡 파트너스 API로 카테고리+월령 키워드 검색
  - 표시 개수: TOP 5~10개
  - 정렬 기준:
    1. `shared_products`의 `purchaseCount` (구매 많은 순)
    2. `shared_products`의 `trackerCount` (관심 많은 순)
    3. 초기(데이터 없을 때) → 쿠팡 파트너스 베스트셀러 기준
  - 캐싱 전략: 동일 카테고리+월령 결과 6시간 Firebase 캐싱 → API 호출 최소화
  - API 제재 방지: 카테고리 탭 연속 클릭 시 디바운싱 적용 (300ms)
- **우선순위**: 🟡 중간 (12번 카테고리 체계 + shared_products 구현 후)

### 14. 상품 개수 늘어날수록 카테고리 영역 작아지는 레이아웃 버그

- **문제/기획 의도**: 상품 리스트가 길어지면 카테고리 필터 영역이 밀려서 작아짐. 카테고리 접근성 저하.
- **현재 상태**: 카테고리 칩 영역이 고정 높이가 아닌 flex로 구성되어 상품 리스트에 밀림.
- **구현 방향**: 카테고리 영역에 고정 높이 부여. 상품 리스트는 별도 `FlatList`/`ScrollView`로 분리하여 독립 스크롤. StickyHeader 적용 검토.
- **우선순위**: 🔴 높음 (UI 버그)

### 15. 상품 카드 삭제 UI 개선

- **문제/기획 의도**: 현재 삭제 방법이 직관적이지 않음. 실수 삭제 방지 + 다양한 삭제 인터랙션 제공.
- **현재 상태**: 삭제 UI 미확인 (기존 지금이야 방식 그대로일 가능성).
- **구현 방향**:
  - 길게 누르기: 카드에 삭제 버튼 오버레이 노출
  - 왼쪽 스와이프: 카드 왼쪽에 빨간 삭제 버튼 슬라이드
  - 삭제 버튼 탭 시: "관심상품에서 삭제하시겠어요?" 확인 알럿
  - 확인 후 삭제 + 해당 상품 알림 자동 중단 (19번 연동)
  - `react-native-gesture-handler` 또는 `Swipeable` 컴포넌트 활용
- **우선순위**: 🟡 중간

### 19. 관심상품 삭제 시 해당 상품 알림 자동 중단

### 22. 구매목록 기반 재구매 알림

- **문제/기획 의도**: 소모품(기저귀/분유/물티슈)은 주기적 재구매가 필요. 자동 소진일 계산으로 재구매 시점 알림.
- **현재 상태**: 미구현. (015_재구매알림.md에 설계만 존재)
- **구현 방향**:

  **소모품 자동 주기 계산:**
  - 기저귀: 상품명에서 수량 추출 (예: "100매")
    - 신생아(0~3개월): 하루 10매
    - 영아(4~12개월): 하루 6~8매
    - 유아(13개월~): 하루 4~6매
    - → 아이 월령 + 수량으로 소진일 자동 계산
  - 분유: 상품명에서 용량 추출 (예: "800g")
    - 0~3개월: 하루 약 600ml → 800g ≈ 약 20일
    - 4~6개월: 하루 약 800ml → 800g ≈ 약 15일
  - 물티슈: 상품명에서 매수 추출 (예: "100매 10팩")
    - 하루 평균 20매 기준으로 소진일 계산

  **재구매 알림 대상 카테고리 (선별 적용):**
  - 알림 있음 (소모품): 기저귀, 분유, 물티슈, 이유식, 유아식, 스킨케어
  - 알림 없음 (내구재): 의류, 신발, 장난감, 유모차/카시트, 가구, 학용품, 전자기기

  **알림 문구 예시:**
  - "하기스 기저귀 100매, 슬슬 다 떨어질 때가 됐어요! 지금 가격 확인해보세요"
  - "압솔루트 분유 800g, 약 3일 후 소진 예정이에요. 지금 최저가로 구매해요!"

  **사용자 직접 설정 옵션:**
  - 자동 계산 외 사용자가 직접 주기 설정 가능
  - 알림 ON/OFF 개별 설정 가능

- **우선순위**: 🟡 중간 (Phase 2. 21번 구매목록 완료 후)

## [핵심 아키텍처 - 반드시 구현]

### shared_products 공유 가격 구조

- **문제/기획 의도**: 아이고의 모든 추천/통계 기능의 근간. 사용자별 개별 저장으로는 "N개월 맘들의 픽", 인기 상품 랭킹 등 통계 기능 구현 불가. 전체 사용자 데이터를 공유 컬렉션에 집계해야 함.
- **현재 상태**: Firestore rules에 `shared_products` 규칙은 정의되어 있으나, 실제 코드에서 읽기/쓰기 로직 미구현. 문서에만 설계 존재.
- **구현 방향**:
  - Firestore 구조:
    ```
    shared_products/{productId}
      - currentPrice: number          // 현재 가격
      - previousPrice: number         // 이전 가격
      - priceHistory: PricePoint[]    // 가격 이력
      - trackerCount: number          // 관심 등록 수
      - purchaseCount: number         // 구매 횟수
      - ageGroup: string              // 월령 구간 (예: "0-6", "7-12")
      - category: BabyCategory        // 카테고리
      - lastCheckedAt: string         // 마지막 가격 체크 시간
    ```
  - `users/{uid}/items/{productId}` 는 목표가/설정/구매이력만 저장
  - 실제 가격 데이터는 `shared_products`에서 공유
  - 상품 등록 시: `shared_products` 문서 생성 + `trackerCount` +1
  - 상품 삭제 시: `trackerCount` -1 (0이 되어도 문서 유지 — 가격 이력 보존)
  - 가격 업데이트: GitHub Actions 가격 체크 봇 (Admin SDK)
  - 왜 반드시 필요한가:
    - 통계 집계 → 월령별 인기 상품 자동 추천
    - API 호출 효율화 → 상품별 1번만 체크 (사용자 수 무관)
    - 13번 카테고리 추천 상품의 정렬 기준 데이터
- **우선순위**: 🔴 높음 (13번, 22번 등 다수 기능의 전제조건)

- **구현 방향**: 현재 로직 유지. Production 빌드 테스트 시 정상 동작 확인 필요. 개발 환경에서는 수동 재시작이 불가피.
