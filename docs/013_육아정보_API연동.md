# 013. 육아정보 API 연동 계획

## 상태: 설계 완료 / API 키 발급 대기

## 현재 구조 (1단계 — 완료)
- `app/(tabs)/babyinfo.tsx`: 앱 내장 정적 데이터 + 외부 링크
- 예방접종: 하드코딩 스케줄 11단계
- 정부 지원금: 정적 카드 6종 → 외부 URL
- 어린이집/유치원: 정적 카드 3종 → 외부 URL

## 목표 구조 (2단계 — API 연동)
- `services/publicApi.ts`: 공공데이터 API 클라이언트 (신규)
- `app/(tabs)/babyinfo.tsx`: API 데이터 기반 동적 렌더링
- 앱 내에서 직접 정보 조회, 외부 이동 최소화

## API 목록

### 1. 예방접종도우미 API (질병관리청)

**포털**: https://www.data.go.kr/data/15001599/openapi.do
**서비스명**: 예방접종도우미_국가예방접종 지정의료기관 현황

| 항목 | 내용 |
|------|------|
| 기관 | 질병관리청 |
| 엔드포인트 | `http://apis.data.go.kr/1790387/vcninfo` |
| 인증 | 서비스 키 (공공데이터포털 발급) |
| 응답 | XML/JSON |
| 활용 | 가까운 예방접종 기관 검색, 접종 가능 백신 목록 |

**추가 활용 가능 API**:
- 예방접종 통합관리 시스템 접종 스케줄 조회
- 영유아 건강검진 시기 조회 (건강보험공단)

### 2. 복지로 API (보건복지부)

**포털**: https://www.data.go.kr/data/15083277/openapi.do
**서비스명**: 복지서비스 정보 조회

| 항목 | 내용 |
|------|------|
| 기관 | 보건복지부 |
| 엔드포인트 | `http://apis.data.go.kr/B554287/LocalGovernmentWelfareInformations` |
| 인증 | 서비스 키 (공공데이터포털 발급) |
| 응답 | XML/JSON |
| 활용 | 지역별 복지 서비스 검색, 수혜 조건/금액 조회 |

### 3. 어린이집 정보 API (보육진흥원)

**포털**: https://www.data.go.kr/data/15004402/openapi.do
**서비스명**: 어린이집 현황 정보

| 항목 | 내용 |
|------|------|
| 기관 | 한국보육진흥원 |
| 엔드포인트 | `http://api.childcare.go.kr` |
| 인증 | 서비스 키 |
| 응답 | XML/JSON |
| 활용 | 주변 어린이집 검색, 정원/현원, 보육료, 평가 등급 |

### 4. 아동수당/양육수당 API (선택)

**포털**: 정부24 API 또는 복지로 API에서 통합 조회
- 아동수당 수급 자격 조회
- 양육수당/부모급여 금액 조회
- 지역별 추가 지원 정보

## 파일 구조 계획

```
services/
├── publicApi.ts          ← 공공데이터 API 클라이언트 (신규)
│   ├── initPublicApi()          — API 키 로드
│   ├── fetchVaccineCenters()    — 예방접종 기관 검색
│   ├── fetchWelfareServices()   — 복지 서비스 조회
│   ├── fetchChildcareCenters()  — 어린이집 검색
│   └── fetchHealthCheckup()     — 영유아 건강검진 시기
│
app/(tabs)/
├── babyinfo.tsx          ← 리팩토링: API 데이터 + 정적 fallback
```

export interface HealthCheckup {
  round: number;          // 차수 (1~8차)
  ageRange: string;       // 대상 월령 (예: "4~6개월")
  items: string[];        // 검진 항목
  isCurrent: boolean;     // 현재 해당 여부

export function getHealthCheckupSchedule(
  babyMonths: number,
): HealthCheckup[] {
  // 정적 데이터 (API 불필요 — 국가 표준 스케줄)

    { round: 2, ageRange: '9~12개월', items: ['문진', '신체계측', '발달선별검사'] },
    { round: 3, ageRange: '18~24개월', items: ['문진', '신체계측', '발달선별검사', '구강검진'] },

    { round: 5, ageRange: '42~48개월', items: ['문진', '신체계측', '발달선별검사', '구강검진', '시력검사'] },
    { round: 6, ageRange: '54~60개월', items: ['문진', '신체계측', '발달선별검사', '구강검진', '시력검사'] },

  ];

  const ranges = [[4,6],[9,12],[18,24],[30,36],[42,48],[54,60],[66,72]];
  return schedule.map((s, i) => ({
    ...s,
    isCurrent: babyMonths >= ranges[i][0] && babyMonths <= ranges[i][1],
  }));
}

## babyinfo.tsx 리팩토링 계획

### 변경 전 (현재)
```
정적 데이터 → UI 렌더링 → 외부 링크 이동
```

### 변경 후 (2단계)
```
1. 정적 데이터 (fallback)
2. API 키 있으면 → API 호출 → 캐싱 → 동적 렌더링
3. API 실패 시 → 정적 데이터 fallback
```

### 섹션별 변경

| 섹션 | 현재 | 변경 후 |
|------|------|---------|
| 예방접종 | 정적 스케줄 | 정적 스케줄 유지 + "가까운 접종기관 검색" 버튼 (위치 기반) |
| 건강검진 | 없음 | 영유아 건강검진 스케줄 추가 (정적, 월령 하이라이트) |
| 정부 지원금 | 정적 6종 | API로 지역별 맞춤 복지 서비스 + 정적 fallback |
| 어린이집 | 외부 링크 | API로 주변 어린이집 검색 + 정원/현원/평가 등급 표시 |

### 캐싱 전략
- `AsyncStorage` 캐싱 (24시간)
- 캐시 키: `publicapi-{endpoint}-{params-hash}`
- API 실패 시 캐시 반환, 캐시도 없으면 정적 데이터

### 위치 기반 기능
- `expo-location`으로 현재 위치 획득 (선택적 권한)
- 위치 없으면 시/도 수동 선택 드롭다운
- 예방접종 기관, 어린이집 검색에 활용
