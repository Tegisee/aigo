# 002. 기술스택 + 폴더구조

## 기본 방침
- 지금이야 코드베이스 fork → 육아 특화 기능만 추가
- 검증된 기술스택 그대로 유지, 변경 최소화
- 쿠팡 파트너스 계정 동일 사용

## 프레임워크
- React Native + Expo SDK 55
- EAS Build 기준 (Expo Go 사용 금지 — Share Intent 미작동)
- TypeScript

## 상태관리
- Zustand + AsyncStorage persist
- 지금이야 대비 추가 상태:
  - `children: ChildProfile[]` — 아이 프로필 (생년월일, 별명)
  - `selectedChildId: string | null` — 현재 선택된 아이
  - `repurchaseItems: RepurchaseItem[]` — 재구매 알림 설정

## 데이터베이스
- Firebase Firestore (클라우드 백업) — **신규 프로젝트** (지금이야와 분리)
- Firebase Anonymous Auth (기기별 고유 ID)
- AsyncStorage (로컬 1차 저장)
- Firebase 초기화 실패 시 AsyncStorage 폴백

## 네비게이션
- Expo Router (파일 기반 라우팅)
- 하단 탭바: 홈 | 설정

## 차트 / 공유 / 알림
- react-native-gifted-charts + react-native-svg (가격 변동 그래프)
- expo-share-intent (쿠팡 공유 URL 수신)
- Expo Push Notifications + FCM V1 (Phase 2)

## 쿠팡 연동
- 쿠팡 파트너스 API (HMAC 서명 + 딥링크) — 동일 계정
- CoupangScraper (WebView DOM 스크래핑)

## 개발 착수 전 필수 교체/설정 항목

### 1. Firebase (신규 프로젝트)
지금이야 Firebase 프로젝트와 데이터가 섞이면 안 되므로 완전 분리 필수.

| # | 작업 | 파일/위치 | 이유 |
|---|------|-----------|------|
| 1-1 | [ ] Firebase 콘솔에서 아이고 신규 프로젝트 생성 | [Firebase Console](https://console.firebase.google.com) | 지금이야와 사용자 데이터·Firestore 컬렉션이 섞이는 것을 방지 |
| 1-2 | [ ] google-services.json 교체 (Android) | `aigo/google-services.json` | 현재 TODO 플레이스홀더 상태 — 실제 Firebase 프로젝트 값 필요 |
| 1-3 | [ ] GoogleService-Info.plist 교체 (iOS) | `aigo/GoogleService-Info.plist` | iOS 빌드 시 Firebase SDK가 이 파일로 초기화함 |
| 1-4 | [ ] firebase.ts 내 config 값 교체 | `services/firebase.ts` | apiKey, projectId, storageBucket, appId 모두 TODO 상태 — 앱 실행 불가 |
| 1-5 | [ ] Firestore 보안 규칙 설정 | Firebase Console → Firestore | 기본 규칙은 30일 후 전체 차단 — 프로덕션용 규칙 필요 |
| 1-6 | [ ] Anonymous Auth 활성화 | Firebase Console → Authentication | 로그인 없이 기기별 사용자 식별에 필수 |
