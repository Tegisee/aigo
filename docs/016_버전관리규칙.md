# 016. 버전 관리 규칙

## 버전/버전코드 관리

### 구조
- **version** (사용자 표시): `{major}.{minor}.{patch}` (예: 1.0.3)
- **versionCode** (Play Console 내부): 정수, 매 업로드마다 반드시 증가
- 설정 위치: `app.config.js` — `expo.version` + `expo.android.versionCode`

### 버전 올리는 기준
| 변경 종류 | version | versionCode |
|-----------|---------|-------------|
| 버그 수정 | patch +1 (1.0.2 → 1.0.3) | +1 이상 |
| 기능 추가 | minor +1 (1.0.3 → 1.1.0) | +1 이상 |
| 대규모 변경 | major +1 (1.1.0 → 2.0.0) | +1 이상 |

### 주의사항
- Play Console은 **versionCode가 이전보다 높아야** 업로드 허용
- EAS 빌드 시 versionCode를 자동 증가시키는 경우가 있음 (app.config.js 값과 다를 수 있음)
- 실제 업로드된 versionCode는 Play Console에서 확인

## 파일명 규칙

```
aigo-v{version}-vc{versionCode}.{aab|apk|ipa}
```

예시:
- `aigo-v1.0.1.aab` (초기 — 버전코드 미표기)
- `aigo-v1.0.3-vc7.aab` (현재 규칙)
- `aigo-v1.1.0-vc10.apk` (디버그용 APK)

저장 위치:
- Android: `~/aigo/builds/android/`
- iOS: `~/aigo/builds/ios/`

## 빌드 전 체크리스트

### 1. 코드 점검
- [ ] `npx tsc --noEmit` — TypeScript 에러 없음 확인
- [ ] 변경 파일 import/export 누락 확인
- [ ] `.env` 환경변수 확인 (EAS Secret 등록 여부)

### 2. 버전 업데이트
- [ ] `app.config.js` — `version` 올림
- [ ] `app.config.js` — `android.versionCode` 올림 (이전 빌드보다 높게)
- [ ] Play Console에서 현재 최대 versionCode 확인 후 설정

### 3. Git 커밋
- [ ] 변경사항 git add + commit
- [ ] commit 메시지에 버전 명시 (예: `chore: v1.0.3 빌드 준비`)
- [ ] `git push origin main`

### 4. 빌드
```bash
# EAS 클라우드 빌드
cd ~/aigo/aigo && eas build --profile production --platform android

# 로컬 빌드 (크레딧 소진 시)
export ANDROID_HOME=~/Library/Android/sdk
export JAVA_HOME=/Library/Java/JavaVirtualMachines/temurin-17.jdk/Contents/Home
cd ~/aigo/aigo && eas build --local --profile production --platform android
```

### 5. 빌드 결과물 관리
- [ ] `.aab` 파일을 `~/aigo/builds/android/`로 이동
- [ ] 파일명을 `aigo-v{version}-vc{versionCode}.aab`로 변경
- [ ] 실제 versionCode 확인: `bundletool dump manifest --bundle=파일.aab | grep versionCode`

## Play Console 업로드 순서

### 1. 비공개 테스트 출시
1. Play Console > 테스트 > 비공개 테스트 > 새 출시 만들기
2. `.aab` 파일 업로드
3. 출시명 입력 (예: `9(1.0.3)`)
4. 출시 노트 작성
5. "검토를 위해 출시 보내기" 클릭

### 2. 검토 완료 후
- 검토 통과 시 자동으로 비공개 테스트 배포
- 테스터 그룹(aigo_app@googlegroups.com)에게 업데이트 제공

### 3. 프로덕션 출시 (비공개 테스트 완료 후)
1. Play Console > 프로덕션 > 새 출시 만들기
2. 비공개 테스트에서 사용한 `.aab` 프로모션
3. 검토 후 배포
