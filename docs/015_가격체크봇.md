## 알림 전략

### 즉시 알림 (6회 모두)
| 조건 | 알림 타입 | 문구 예시 |
|------|-----------|-----------|
| 목표가 도달 | target_reached | "아이고, 지금이 기회! 하기스 기저귀 38,900원 — 목표가 도달!" |
| 가격 하락 | price_drop | "아이고~ 가격 떨어졌다! 45,000원 → 38,900원!" |
| 역대 최저 | lowest_ever | "역대 최저가! 38,900원 — 지금까지 가장 낮은 가격이에요!" |
| 최저가 (목표 없음) | lowest_no_target | "최저가 갱신! 38,900원" |

### 21시 전용 알림 (하루 1회)
| 조건 | 알림 타입 | 문구 예시 |
|------|-----------|-----------|
| N일 무변동 (2일+) | no_change | "N일째 가격 변동이 없어요. 슬슬 구매하실 때가 됐을까요?" |
| 재구매 D-3 이내 | repurchase | "하기스 기저귀, 슬슬 다 떨어질 때가 됐어요!" |

## 데이터 저장 vs 알림 분리

| 동작 | 실행 시점 | 설명 |
|------|-----------|------|
| **Firestore 가격 저장** | 6회 모두 | 가격 그래프용 — 항상 currentPrice + priceHistory 업데이트 |
| **shared_products 업데이트** | 6회 모두 | 공유 가격 데이터 — 인기 상품/통계용 |
| **푸시 알림** | 조건별 분기 | 즉시(하락/목표/최저) + 21시(무변동/재구매) |
| **유저 정리** | 21시만 | 만료 토큰 제거 + 30일 비활성 유저 삭제 |

## Firebase 서비스 계정 발급 방법

1. Firebase Console → aigo-a → 프로젝트 설정 (톱니바퀴)
2. 서비스 계정 탭
3. "새 비공개 키 생성" 클릭
4. JSON 파일 다운로드
5. 파일 내용 전체를 GitHub Secret `FIREBASE_SERVICE_ACCOUNT_KEY`에 등록
6. 다운로드한 JSON 파일은 삭제 (보안)
