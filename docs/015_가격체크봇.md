## 알림 전략

### 즉시 알림 (6회 모두)
| 조건 | 알림 타입 | 문구 예시 |
|------|-----------|-----------|
| 목표가 도달 | target_reached | "아이고, 지금이 기회! 하기스 기저귀 38,900원 — 목표가 도달!" |
| 가격 하락 | price_drop | "아이고~ 가격 떨어졌다! 45,000원 → 38,900원!" |
| 역대 최저 | lowest_ever | "역대 최저가! 38,900원 — 지금까지 가장 낮은 가격이에요!" |
| 최저가 (목표 없음) | lowest_no_target | "최저가 갱신! 38,900원" |

### 21시 전용 알림 (하루 1회)
| 조건 | 알림 타입 | 문구 예시 |
|------|-----------|-----------|
| N일 무변동 (2일+) | no_change | "N일째 가격 변동이 없어요. 슬슬 구매하실 때가 됐을까요?" |
| 재구매 D-3 이내 | repurchase | "하기스 기저귀, 슬슬 다 떨어질 때가 됐어요!" |
| 예방접종 미접종 | vaccine_overdue | "[아이이름]의 [백신명] 접종 시기가 지났어요! 확인해보세요" |

### 파트너스 API 구조적 한계 (04-14 확인)
- 파트너스 검색 API는 **키워드 검색만 지원**, productId 기반 조회 API 없음
- 검색 결과의 productId는 파트너스 풀 기준이라 사용자가 공유한 상품 URL의 productId와 불일치 빈번
- 실측 매칭률: 7개 상품 중 4개 성공 (57%), 3개 파트너스 풀에 없어서 스킵
- 서버사이드 스크래핑(Puppeteer) 전환 검토 → 쿠팡이 GitHub Actions IP 차단 중 (지금이야에서 이미 확인) → 불가
- **현재 결정**: 파트너스 API 유지, 매칭 실패 상품은 안전하게 스킵 (앱 포그라운드 CoupangScraper가 보충)

## Firebase 서비스 계정 발급 방법

1. Firebase Console → aigo-a → 프로젝트 설정 (톱니바퀴)
2. 서비스 계정 탭
3. "새 비공개 키 생성" 클릭
4. JSON 파일 다운로드
5. 파일 내용 전체를 GitHub Secret `FIREBASE_SERVICE_ACCOUNT_KEY`에 등록
6. 다운로드한 JSON 파일은 삭제 (보안)
