# 010. 상품 정보 스크래핑 + 카테고리 분류

## 스크래핑 방식: WebView DOM 파싱
- 쿠팡은 서버사이드 렌더링 + 클라이언트 하이드레이션 → 순수 fetch로는 가격 추출 실패 빈번
- 해결: 숨겨진 WebView에서 페이지 로드 → JS 인젝션으로 DOM에서 추출

## 구현 파일
| 파일 | 역할 |
|------|------|
| `components/CoupangScraper.tsx` | WebView 기반 DOM 파싱 (상품 등록 + 수동 새로고침) |
| `components/PriceChecker.tsx` | 포그라운드 자동 가격 체크 (큐 방식 순차 처리) |
| `services/productMeta.ts` | fetch 기반 OG 태그 파싱 (보조 수단) |

## CoupangScraper 동작
1. WebView에 쿠팡 URL 로드
2. 딥링크 차단 JS (`BLOCK_DEEPLINK_JS`) 선행 주입
3. 상품 페이지 도달 감지 (`/vp/products/` 또는 `/vm/products/`)
4. DOM 파싱 JS (`SCRAPE_JS`) 인젝션
5. 결과: `{ title, price, image, resolvedUrl }`
6. 가격+이미지 있으면 성공, 없으면 단계적 재시도 (2초/4초/6초)
7. 20초 타임아웃 → 에러 콜백

## 추출 우선순위

### Title
1. `og:title` meta
2. `.prod-buy-header__title` (h2/h1)
3. `<title>` 태그

### Price
1. `.total-price strong`
2. `.prod-sale-price .total-price`
3. `product:price:amount` meta
4. JSON-LD `offers.price`

### Image
1. `og:image` meta
2. `.prod-image__detail img`

## PriceChecker (포그라운드 가격 체크)
- 앱 열릴 때 1회 실행 (2분간 active)
- 큐 방식: 등록 상품 순차 체크
- fetch HTML → WebView에 직접 로드 (Universal Link 우회)
- 가격 변동 감지 시 로컬 푸시 알림
- **현재 비활성화** — 파트너스 API 승인 후 재활성화 예정

## 카테고리 자동 분류
- 상품 등록 시 `classifyCategory(productName)` 적용
- 키워드 매칭 기반: 기저귀, 분유, 물티슈, 유모차/카시트, 수유용품, 의류, 장난감, 가구, 기타
- 브랜드명 포함 (하기스, 보솜이, 팸퍼스, 남양, 매일, 피셔프라이스 등)
- 사용자 수동 변경 가능 (드롭다운)
